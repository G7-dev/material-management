# 库存管理逻辑修复说明

## 问题描述

用户提问：提交申请前订书机迷你型有3个，提交完申请后，又删除申请单后，库存应该恢复成3个还是0个？

**期望答案**：应该恢复成3个

## 原有问题

### 错误逻辑
```
提交申请 → 立即扣减库存 → 审批通过/拒绝 → 删除申请 → 库存不恢复
```

### 问题分析
在 `DailyCollection.tsx` 中，提交申请时（handleSubmit）立即扣减了库存：
```typescript
// 提交时立即扣减库存（错误！）
const newStock = Math.max(0, currentStock - quantity);
updateItemStock(targetItem.id, newStock);
```

这导致：
- 提交申请后，库存就被扣减
- 如果删除申请单（无论是否审批），库存不会恢复
- 不符合实际业务流程

## 修复后的正确逻辑

### 正确流程
```
提交申请 → 库存不变（待审核）→ 审批通过 → 扣减库存 → 删除申请（已批准）→ 恢复库存
                    ↓ 审批拒绝
                  库存不变 → 删除申请（已拒绝）→ 库存不变
```

### 修改内容

#### 1. 提交申请时不扣减库存（DailyCollection.tsx）
```typescript
const handleSubmit = () => {
  // ... 保存申请记录 ...
  saveApplicationRecord({...});
  
  // 删除库存扣减逻辑（重要！）
  // ❌ 删除以下代码：
  // const newStock = Math.max(0, currentStock - quantity);
  // updateItemStock(targetItem.id, newStock);
}
```

#### 2. 审批通过时扣减库存（需要在审批管理页面添加）
```typescript
// 在审批管理页面，当点击"通过"时：
const handleApprove = (recordId: string) => {
  // 1. 更新申请状态为 approved
  updateApplicationStatus(recordId, 'approved');
  
  // 2. 扣减库存
  const record = getApplicationRecords().find(r => r.id === recordId);
  if (record) {
    const inventoryItems = getAllInventoryItems();
    const item = inventoryItems.find(i => i.name === record.itemName);
    if (item) {
      const newStock = Math.max(0, item.stock - record.quantity);
      updateItemStock(item.id, newStock);
    }
  }
}
```

#### 3. 删除申请单时不恢复库存（ApplicationRecords.tsx）
```typescript
const handleDelete = (id: string) => {
  // 删除申请记录（不再恢复库存）
  deleteApplicationRecord(id);
  setRecords(getApplicationRecords());
  toast.success('申请单已删除');
};
```

**注意**：删除申请单不恢复库存，因为：
- 待审核状态：从未扣减库存，无需恢复
- 已批准状态：库存已在审批时扣减，删除申请单不代表归还物品
- 已拒绝状态：从未扣减库存，无需恢复

## 业务流程对比

### 修复前（错误）

|------|---------|------|
| 提交申请（3个→1个） | -2个 | 不应扣减 |
| 删除申请 | 不变（1个） | 应该恢复 |
| 最终结果 | 1个 | ❌ 错误 |

### 修复后（正确）
| 步骤 | 库存变化 | 说明 |
|------|---------|------|
| 提交申请（3个） | 不变 | 待审核，不扣减 |
| 审批通过（3个→1个） | -2个 | 实际领用，扣减 |
| 删除申请（1个） | 不变 | 已领用，不恢复 |
| 最终结果 | 1个 | ✅ 正确 |

## 重要说明

### 为什么删除申请单不恢复库存？

**原因**：
1. **审批通过** = 物品已实际领用
2. **删除申请单** ≠ 归还物品
3. 如果物品已领用，需要走"归还"流程，而不是删除申请单

**正确做法**：
- 物品领用后需要归还：走物品归还流程
- 申请单填写错误：删除申请单，但不影响已领用物品
- 库存管理基于实际物品流动，而非申请单存在与否

### 业务流程建议

1. **提交申请**：仅记录申请信息，不操作库存
2. **审批流程**：
   - 通过：扣减库存，记录领用
   - 拒绝：仅更新状态，不操作库存
3. **物品归还**：
   - 增加库存数量
   - 记录归还信息
4. **申请单管理**：
   - 删除申请单不影响库存
   - 仅作为记录管理

## 回答用户问题

**用户问**：提交申请前订书机迷你型有3个，当提交完申请后，又删除申请单后，库存应该恢复成3个的显示还是0个

**回答**：
- 如果审批通过后再删除：库存显示1个（实际领用了2个）
- 如果审批拒绝后删除：库存显示3个（未领用）
- 如果待审核时取消/删除：库存显示3个（未领用）

**关键**：库存变化发生在审批通过时，而不是提交申请时。删除申请单不恢复库存，因为库存已经在审批通过时实际扣减。
