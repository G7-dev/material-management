# 测试删除功能

## 功能说明

已实现以下删除功能：

### 1. 删除单个规格
- 在物品补货管理页面的编辑模态框中
- 每个规格右上角有删除按钮（当规格数量>1时显示）
- 点击后弹出确认对话框
- 确认后从数据库的 sizes JSON 数组中移除该规格
- 自动重新计算总库存
- 刷新界面数据

### 2. 删除整个物品
- 在编辑模态框底部有"删除整个物品"按钮
- 点击后弹出确认对话框，显示警告信息
- 确认后执行软删除（将 status 设为 'inactive'）
- 刷新界面数据

### 3. 日常领用端隐藏无库存规格
- 已修复日常领用端显示逻辑
- 库存为0的规格会自动隐藏
- 显示提示"X个规格暂时缺货"

## 测试步骤

### 测试删除单个规格

1. 打开物品补货管理页面（低库存预警）
2. 找到一个有多规格的物品（如白板笔）
3. 点击"编辑"按钮
4. 在编辑模态框中，找到要删除的规格（如"黑/红/蓝"）
5. 点击该规格右上角的垃圾桶图标
6. 确认删除对话框显示正确的规格名称
7. 点击"确认删除"
8. 验证：
   - 对话框关闭
   - 编辑模态框关闭
   - 列表刷新，该规格已消失
   - 总库存重新计算

### 测试删除整个物品

1. 打开物品补货管理页面
2. 点击任意物品的"编辑"按钮
3. 滚动到模态框底部
4. 点击"删除整个物品"按钮
5. 确认删除对话框显示警告信息
6. 点击"确认删除"
7. 验证：
   - 对话框关闭
   - 编辑模态框关闭
   - 列表刷新，该物品已消失

### 测试日常领用端隐藏规格

1. 打开日常领用页面
2. 找到一个有多规格的物品
3. 点击"查看各规格库存"
4. 验证：
   - 只显示库存 > 0 的规格
   - 库存为0的规格不显示
   - 如果所有规格都无库存，显示"暂无库存"

## 注意事项

1. 当物品只有一个规格时，删除按钮会隐藏（至少保留一个规格）
2. 删除规格是不可逆操作，请谨慎使用
3. 删除整个物品是软删除，数据仍在数据库中（status='inactive'）
4. 所有删除操作都会清除缓存并触发库存更新事件
