# TypeScript 类型错误修复总结

## 问题概述

在 `LowStockAlert.tsx` 文件中存在多个 TypeScript 类型错误，主要是因为 `Material` 接口缺少运行时动态添加的属性。

## 错误类型

### 🔴 严重错误（20+ 个）
1. **类型 "Material" 上不存在属性** - `currentThreshold`, `currentStock`, `spec` 等
   - 这些属性是在运行时通过 `enriched` 计算添加的
   - TypeScript 编译时无法识别这些动态属性

2. **"selectedSize" 可能为 null**
   - 在使用 `selectedSize.id` 时没有做空值检查

### 🟡 警告错误（7 个）
1. 未使用的导入（`AppSelect`, `format`）
2. 未使用的变量（`CATEGORIES`, `UNITS`, `hasSizes`, `override`, `newStock`）

## 修复方案

### 1. 创建扩展类型接口

```typescript
// 扩展 Material 类型，添加运行时计算属性
interface MaterialWithStatus extends Material {
  currentThreshold: number;
  severity: StockSeverity;
  currentStock: number;
  lowStockSizes: MaterialSize[];
  spec?: string;
  lastRestock?: string;
}
```

**原因**：明确声明所有运行时属性，让 TypeScript 知道这些属性的存在

### 2. 类型断言

```typescript
const enriched = useMemo(() =>
  materials.map(material => {
    const threshold = getThreshold(material);
    const status = getMaterialStockStatus({ ...material, safe_stock: threshold });
    
    return {
      ...material,
      currentThreshold: threshold,
      severity: status.severity,
      currentStock: status.overallStock,
      lowStockSizes: status.lowStockSizes,
    } as MaterialWithStatus;  // 类型断言
  }),
  [materials, thresholds]
);
```

**原因**：告诉 TypeScript 返回的对象是 `MaterialWithStatus` 类型

### 3. 更新所有组件 Props

```typescript
// 之前
interface RestockModalProps {
  item: Material;
  ...
}

// 修复后
interface RestockModalProps {
  item: MaterialWithStatus;
  ...
}
```

**原因**：确保组件接收正确类型的数据

### 4. 空值安全检查

```typescript
// 之前
onConfirm(item.id, selectedSize.id, numQty);

// 修复后
onConfirm(item.id, selectedSize?.id || null, numQty);
```

**原因**：防止 `selectedSize` 为 null 时访问 `id` 属性导致运行时错误

### 5. 提供默认值

```typescript
// 之前
{item.lastRestock}

// 修复后
{item.lastRestock || '-'}
```

**原因**：处理可选属性可能为 undefined 的情况

### 6. 清理未使用的代码

```typescript
// 移除未使用的导入
import { AppSelect } from '../components/ui/app-select';  // 删除
import { format } from 'date-fns';  // 删除

// 使用下划线标记未使用的参数
const handleSaveEdit = (
  materialId: string,
  _override: ItemOverride,  // 标记未使用
  _newStock: number,         // 标记未使用
  newThreshold: number | Record<string, number>
) => { ... }
```

**原因**：减少代码体积，提高可维护性
