## 🎯 功能特性

### ✅ 删除整个物品
- 从 `materials` 表中永久删除记录
- 所有规格数据被删除
- 无法恢复
- 刷新后不再出现

### ✅ 删除单个规格
- 从 `sizes` JSON 数组中移除
- 重新计算总库存
- 无法恢复

### ✅ 批量删除规格
- 选择多个规格批量删除
- 全选/取消全选功能
- 至少保留一个规格

### ✅ 编辑模态框优化
- 当前库存设为只读
- 阈值调整功能正常
- 提示用户使用"补货"功能调整库存

## 📊 数据库影响

### 被删除的数据
- 物品基本信息（名称、分类、规格等）
- 库存信息（当前库存、阈值）
- 规格信息（JSON数组中的数据）
- 物品图片URL

### 保留的数据
- 库存流水记录（`inventory_logs`）- 用于审计
- 申请记录（`requisitions`）- 用于报表

## ⚠️ 重要警告

### 不可逆操作
- 删除后无法恢复
- 没有回收站功能
- 删除前请仔细确认

### 数据保留策略
- 保留审计日志（库存流水）
- 保留申请历史
- 物品数据永久删除

## 📝 测试建议

### 测试删除整个物品
1. 打开物品权限管理页面
2. 点击物品的垃圾桶图标
3. 点击"删除整个物品"
4. 验证：
   - [ ] 物品从列表中消失
   - [ ] 刷新页面后仍然不出现
   - [ ] 数据库中该记录已不存在

### 测试删除规格
1. 打开物品权限管理页面
2. 点击多规格物品的垃圾桶图标
3. 选择1个或多个规格
4. 点击"删除选中规格"
5. 验证：
   - [ ] 选中的规格被删除
   - [ ] 未选中的规格保留
   - [ ] 总库存重新计算

### 测试硬删除效果
1. 使用 Supabase SQL Editor
2. 查询被删除的物品ID
3. 验证：
   ```sql
   SELECT * FROM materials WHERE id = 'deleted-item-id';
   ```
4. 结果应为：0 行
