# 物品上架功能修复说明

## 问题描述
用户在物品上架页面填写完所有信息后，点击"确认上架"按钮无反应，无法完成物品上架操作。

## 问题原因
代码中存在一个隐藏的必填字段 `组件库存台`（`stockPlatform`），但在表单中没有提供输入控件，导致用户无法填写该字段，进而无法通过表单验证。

## 修复内容

已在 `src/app/pages/ItemUpload.tsx` 中添加：

1. **在库存信息区域添加"组件库存台"选择框**
   - 位置：库存信息卡片内，上架数量和有效期限下方
   - 类型：下拉选择框
   - 选项：
     - 备件库存
     - 耗材库存
     - 办公用品库
     - 设备库
     - 应急物资库
     - 其他

2. **更新表单数据结构**
   - 在 `EMPTY_FORM` 初始化数据中添加 `stockPlatform: ''`

## 验证修复

修复后，物品上架页面的**库存信息**区域将显示"组件库存台"选择框，用户必须选择库存位置后才能上架物品。

## 使用说明

1. 打开物品上架页面
2. 填写物品基本信息（名称、分类等）
3. 在"库存信息"区域：
   - 填写上架数量
   - 选择有效期限（可选）
   - **选择组件库存台（必填）**
   - 设置低库存预警数量（可选）
4. 点击"确认上架"按钮

## 注意事项

- "组件库存台"为必填项，必须选择后才能上架
- 选择后会在右侧预览卡片中显示
- 该字段会保存在本地存储中，供其他页面使用
